# Frontend Conventions

## 한국어

- React + TypeScript strict mode를 유지합니다.
- 화면 기준으로 `pages`, `components`, `api`, `types` 확장을 고려합니다.
- API 응답 타입을 명시합니다.
- 임시 `any` 사용을 피합니다.
- 폼 상태와 서버 상태를 분리해서 생각합니다.

## English

- Keep React and TypeScript in strict mode.
- Grow the structure around `pages`, `components`, `api`, and `types`.
- Declare API response types explicitly.
- Avoid temporary `any`.
- Treat form state and server state as separate concerns.
